## 2025-02-23 - Optimizing Federated Grid Discovery with Lazy Loading
**Learning:** In the Tunecamp static frontend, dynamic lists rendered across federated network endpoints (e.g., discovery grids and network activity) cause immense network bottlenecks by instantly requesting image assets for unbounded results off-screen. However, universally applying `loading="lazy"` blindly penalizes Largest Contentful Paint (LCP) for above-the-fold content.
**Action:** Implemented a conditional check on the array iteration index (`i < N ? 'eager' : 'lazy'`) inside the template literals of array maps (e.g., `events.map((e, i) => { ... })`). This preserves LCP by keeping the first few top-row images eager, while aggressively lazy-loading everything else. Apply this index-based loading threshold any time unbounded DOM elements with media are dynamically generated.
